1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
	Использовать команду git checkout -- <file> или git restore <file>. 

2.Как вернуть файл из индекса в рабочий каталог?
	Использовать команду git restore --staged <file>.

3.Как отменить ошибочный commit ?
	Использовать команду git revert <commit>

4.Как создать новую ветку и переключится в нее?
	использовать команду git checkout -b <branch>

5.В чем отличение между fast-forward merge  and non fast-forfard merge?
	Отличие этих команд в способе интеграции изменений из одной ветки в другую.

6.Когда может возникнуть конфликт слияния веток, как  его разрешить?
	Конфликт возникает, когда в двух ветках была изменена одна и та же строка в файле или когда некий файл удален в одной ветке и отредактирован в другой. 
	Как правило, конфликты возникают при работе в команде.
	Fast-forward merge происходит, когда ветка, в которую происходит слияние (называемая "вторичной веткой"), не имеет новых коммитов после точки 
	разделения сливаемых веток. В этом случае Git просто перемещает указатель вторичной ветки на последний коммит первичной ветки, делая слияние 
	быстрым и линейным.
	Non fast-forward merge происходит, когда ветка, в которую происходит слияние, имеет новые коммиты после точки разделения. В этом случае Git 
	создает новый коммит слияния, который объединяет изменения обеих веток. Это создает дополнительный коммит слияния в истории проекта.
	
7.Можно ли удалить ветку? Как? 
	Команда для удаления локальной ветки в Git:

	git branch -d  local_branch_name
	git branch – команда для удаления локальной ветки.
	-d – флаг, опция команды git branch, сокращенный вариант записи --delete. Как и следует из названия, предназначен для удаления ветки.
	local_branch_name – имя удаляемой ветки.
